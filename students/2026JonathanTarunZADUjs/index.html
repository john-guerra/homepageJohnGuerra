ZADU.js — Measuring and Visualizing Distortions in Dimensionally Reduced Data on the Browser

Jonathan Tarun Rajasekaran, John A. Guerra-Gomez
Master of Science Thesis, Khoury College of Computer Sciences, Northeastern University, April 2026

Abstract

We present ZADU.js, an open-source JavaScript library that computes per-point Trustworthiness (T) and Continuity (C) scores for dimensionality reduction (DR) embeddings entirely in the browser, without server infrastructure. DR projections such as UMAP introduce distortions like false neighbors and missing neighbors, yet analysts rarely have tools that reveal where and how severely a given embedding misleads them. Existing DR quality frameworks depend on server-side Python environments, making them impractical for browser-based visual analytics. ZADU.js addresses this gap by providing per-point scores that map to visual channels such as color on a scatter plot, letting analysts localize distortion spatially rather than rely on a single aggregate statistic.

We benchmark the metrics on a 200-sample MNIST subset (784 dimensions to 2D), observing baseline scores of T = 0.93 and C = 0.95, and characterize the trade-offs across the parameter space. Metric computation completes in under 25ms for point-level metrics and under 250ms for cluster-level metrics, establishing the feasibility of interactive browser-native quality assessment.

This thesis contributes (1) ZADU.js, an open-source JavaScript library implementing four complementary per-point DR quality metrics; (2) an interactive UMAP hyperparameter tuning application supporting parameter comparison and point-level neighborhood tracking across embeddings; and (3) an end-to-end browser-native deployment that removes the Python and server dependencies of prior DR quality tools.

Problem Statement

Dimensionality reduction algorithms such as UMAP and t-SNE are routinely used to compress high-dimensional data into 2D scatter plots that analysts inspect for clusters, trends, and outliers. Every projection necessarily discards information, and the resulting layout implies geometric relationships that may or may not reflect the actual structure of the original data. The critical question is not whether distortion is present — it always is — but which relationships are distorted and where.

Existing quality assessment libraries operate server-side in Python environments, making them inaccessible to browser-based visualization workflows. Even when quality metrics are computed, they are typically reported as a single global score that cannot reveal the spatial distribution of distortion across the embedding. ZADU.js closes both gaps: it runs natively in the browser and produces per-point scores that can be color-encoded directly onto the scatter plot, transforming a single scalar into a spatial distortion map.
